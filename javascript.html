Using document.getElementById() and .innerHTML

You use the document.getElementById to grab a specific element and do something with it. In the HTML you have an element that has an id like:

<p id="replace">this is some text</p>

NOTE: If you look at the source for the above paragraph, you'll see that I used an "xmp" tag to get the html elements to show up as litterals. But it placed the text way to the left... so I used the style for the p tag on the xmp tag and it worked.

In a javascript file (.js), you use that id to identify the object you're going to do something with.

What the .js code looks like:

var myQuote = '"What is a Week End?"';

var myComment = "Who said this:";

var message = myComment + " " + myQuote;

var paragraphRef = document.getElementById("replace");

paragraphRef.innerHTML = message;

Here's the p called "replace"

In the .html file it's just like the one above (says "this is some text"), but once rendered it gets it's text (innerhtml) from the test.js file:

this is some text

Modernizer

We learned about "Mondernizer". It's a JavaScript library that can tell if a client has all of the CSS3 and HTML5 features you're using. I *think* it let's you make if else statements for browsers that don't support the features you're using. We put the modernizer-latest.js file directly into the root folder for some reason and called it from the html file like this:

<script type="text/javascript" src="javascript/test.js"></script>
